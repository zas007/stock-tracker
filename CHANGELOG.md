## v11.7 — 2026/06/08

### 修正

- **ETF 過濾漏洞（`auto_update_sector_map` / `unknown_codes`）**
  * 舊邏輯：只過濾「含字母或長度 > 4」，`0052` 等 4 碼 `00` 開頭的 ETF 會被補入族群表
  * 新邏輯：兩處都改用 `is_etf_code(code)` 判斷，涵蓋含字母、`00` 開頭、長度 > 4 三種情況
  * 影響：`auto_update_sector_map`、`update_sector_sheet` 的 `unknown_codes`

- **`find_trading_day` 時間門檻 14:00 → 16:30**
  * 舊問題：16:30 後手動執行，T86 資料已釋出，但 14:00 門檻早已通過，部分情境仍退回前一交易日
  * 新邏輯：`now.hour < 16 or (now.hour == 16 and now.minute < 30)` 才退一天
  * 新增執行時間顯示：Step 1 開頭印出 `執行時間：YYYY/MM/DD HH:MM`

- **`load_cache` 誤導性訊息**
  * 舊訊息：`✅ 快取命中（日期：XXXXXX）`，語意像是快取已被採用
  * 新訊息：`📋 XXXXXX 快取讀取成功`，日期前置，語意改為「讀到快取」

- **`config.py` 2897 代號衝突**
  * 工程EPC 族群誤用 `2897`（實為王道銀行），正確代號應為 `9933`（中鼎工程）
  * 移除工程EPC 的 `2897`，CODE_NAME_MAP 同步移除重複的 `2897: 中鼎`

### 調整

- **推薦評分體系重整**
  * `_score_matrix` 滿分 40 → 45，高/中/低集中度差距拉大：
    - 🔵 高度集中：28/34/40 → 30/38/45（連續 ≤3 / ≤7 / >7 天）
    - 🟦 中度集中：16/22/28 → 18/25/32
    - ⬜ 偏低：4/8/12 → 5/9/13
  * `_score_risk` 滿分 20 → 15（🟢低→15，🟡中→7）
  * `_score_volume_ratio` 量比 None 改給基礎分 5（舊為 0），避免快取命中時量比全 None 導致分數扁平；縮量得分 0 → 2
  * 總分上限維持 100

- **推薦過濾新增現價上限**
  * 現價 > 400 元不列入推薦池（價格過高，散戶參與度低）

- **文件整理**
  * `WORKFLOW.md` 刪除（內容整合至 `專案備忘錄.md` 和 skill）
  * `README.md` 大幅精簡，只保留 GitHub 首頁基本說明
  * `專案備忘錄.md` 新增「開新對話時」區塊，補入 raw 連結與對話結束規範

  * 舊問題：族群聯動（Step 5）對所有成員一律呼叫 `fetch_stock_quote()`，但 Step 2 已抓過買超榜的股票，等於同一支股票打兩次 API

  * 修法：`fetch_stock_day()` 改為同時計算漲跌幅（API 回傳整月資料，對前一行收盤計算），回傳值由 3 個擴充為 4 個 `(avg, close, volume, pct_str)`

  * 效果：族群聯動所有成員（不論是否在買超榜）均能正確顯示漲跌幅；對照分析漲跌幅欄也同步受益

  * 舊問題：`current_prices` 只有今日買賣超榜的代號，歷史上曾出現但今日不在榜的股票，對照分析的現價和籌碼集中度欄位會是空白
  * 新邏輯：`update_analysis()` 執行前，先從歷史紀錄收集所有曾出現的代號，過濾出快取缺少的部分，補抓後再進行統計
  * 效果：對照分析現價欄不再有空白，籌碼集中度計算更完整

- **對照分析排序邏輯錯誤**

  * 新邏輯：第一鍵最近出現日降序（最新的在上），同日再依合計累積天數升序（天數少的在上）
